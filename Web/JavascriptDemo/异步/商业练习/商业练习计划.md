# 商业练习计划（异步，1 天版）

目标：在 1 天内完成 3 个独立小页面（库存表、订单列表、付款流水），每个页面使用公开 API + 纯 HTML/CSS/JS；最后整合为一个综合页面。

## 总体原则
- 每个页面都包含：加载中、错误提示、空数据提示
- 数据获取使用 `fetch` + `async/await`
- DOM 渲染使用原生 JS
- 先独立，再整合

## 练习 1：库存表（公开 API）
- API：`https://dummyjson.com/products`
- 文档：`https://dummyjson.com/docs/products`
- 重点字段：`title`、`price`、`stock`、`category`
- 功能：
  - 渲染表格
  - 按库存区间筛选（例如：`<20` / `20-50` / `>50`）
  - 关键字搜索（商品名）

## 练习 2：订单列表（公开 API）
- API：`https://dummyjson.com/carts`
- 文档：`https://dummyjson.com/docs/carts`
- 重点字段：`id`、`total`、`totalProducts`、`totalQuantity`、`discountedTotal`
- 功能：
  - 渲染列表
  - 按订单金额排序（升/降）
  - 显示订单状态（本地规则模拟）

## 练习 3：付款流水（公开 API）
- API：`https://dummyjson.com/transactions`（若不可用改用 `https://dummyjson.com/products` 生成模拟流水）
- 文档：`https://dummyjson.com/docs/transactions`
- 重点字段：`amount`、`date`、`type`
- 功能：
  - 渲染流水
  - 过滤类型（收入/支出）
  - 统计本页合计

## 最终整合
- 单页面分区：库存 / 订单 / 流水
- 顶部导航快速定位
- 复用加载、错误处理逻辑

## 产出
- 3 个独立 HTML 文件
- 1 个整合 HTML 文件

## 1 天执行时间表（约 6-8 小时）
- 09:00-10:30 库存表：拉取数据、渲染表格、基础样式
- 10:30-11:30 库存表：搜索 + 库存筛选 + 状态提示
- 11:30-12:30 订单列表：拉取数据、渲染列表

- 14:30-15:30 付款流水：拉取/生成数据、渲染流水

- 16:30-18:00 整合页：三块合并 + 导航 + 统一加载/错误逻辑
