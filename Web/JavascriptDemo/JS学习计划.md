# JS 学习计划（12 周，全栈入门，3–5 小时/周）

假设每周 2–3 次、每次 60–90 分钟，设备具备 Node 18+、浏览器和 VSCode。

## 总体策略
- 70% 学习 + 30% 练习；每周至少产出 1 个可运行成果。
- 里程碑：第 6 周前端 Todo 可用；第 9 周完成可调用的 REST API；第 12 周交付可展示全栈项目。
- 复盘：每周写 5 点收获 + 1 个踩坑；每两周做一次小迭代。

## 周计划与产出
- **第 1 周：语言基础**  
  - 内容：类型、作用域、函数、条件/循环、`let/const`。  
  - 练习：FizzBuzz、数组求和、字符频次统计。  
  - 产出：1 页语法速查笔记 + 3 段代码。
- **第 2 周：数组与对象**  
  - 内容：`map/filter/reduce/find`、对象遍历、解构、模板字符串。  
  - 练习：学生成绩统计脚本。  
  - 产出：带注释的小脚本。
- **第 3 周：异步思维**  
  - 内容：事件循环、Promise、`async/await`、错误处理。  
  - 练习：fetch 公共 API 打印 JSON；宏/微任务顺序实验。  
  - 产出：实验代码 + 说明。
- **第 4 周：DOM 基础**  
  - 内容：DOM 查询、修改文本/样式、事件基础、事件委托。  
  - 练习：点击计数、输入字数、列表点击高亮。  
  - 产出：小页面 demo（可放在 `第 5 周/day2` 路径旁）。
- **第 5 周：表单与本地存储**  
  - 内容：表单校验、阻止默认、`localStorage` 序列化/恢复。  
  - 练习：Todo MVP（添加/删除/完成状态、计数、持久化）。  
  - 产出：Todo MVP 页面。
- **第 6 周：前端小项目 1.0**  
  - 内容：过滤视图、空状态、loading/错误提示、基础样式。  
  - 练习：打磨 Todo，写 README（安装、用法、截图）。  
  - 产出：可展示的 Todo 前端。
- **第 7 周：模块化与包管理**  
  - 内容：ESM、`import/export`、npm scripts、快速打包（vite 或 parcel）。  
  - 练习：拆分 Todo 为多文件模块，新增 `npm run dev/build`。  
  - 产出：模块化结构 + 构建命令。
- **第 8 周：Node.js 基础**  
  - 内容：CommonJS vs ESM、fs/path、环境变量。  
  - 练习：Node 脚本批量重命名文件或读取日志。  
  - 产出：实用小脚本 + README。
- **第 9 周：REST API**  
  - 内容：Express/Koa 路由、中间件、错误处理、日志。  
  - 练习：实现 Todo/用户 API（列表、创建、更新、删除）。  
  - 产出：可启动的 API 服务（含示例 curl）。
- **第 10 周：数据库接入**  
  - 内容：SQLite（推荐 better-sqlite3）或 MongoDB 选一。  
  - 练习：API 接 DB 完成 CRUD，写初始化/迁移脚本。  
  - 产出：DB 初始化脚本 + 更新后的 API。
- **第 11 周：全栈项目集成**  
  - 内容：前后端联调，错误/空状态、鉴权简版（token 或 mock）。  
  - 练习：记账本/习惯打卡/书签管理任选，接入后端。  
  - 产出：联调成功的全栈应用。
- **第 12 周：部署与展示**  
  - 内容：部署到 Render/Netlify/Cloudflare Pages + Serverless；性能与可维护性复查。  
  - 练习：线上环境健康检查、日志查看、异常兜底页面。  
  - 产出：线上 Demo + 完整 README（安装/运行/部署/改进清单）。

## 测试与验证
- 语言/异步：3–5 个 Jest/Vitest 单测覆盖数组方法与 Promise 链；事件循环实验输出符合预期。  
- Todo 前端：手动测试添加/删除/过滤；刷新后状态保持；表单校验提示。  
- API：用 `supertest`/`curl` 覆盖 200/400/404/500；并行 20 次请求验证幂等。  
- 数据库：初始化脚本可重复执行；空库启动成功；简单迁移后数据仍可读。  
- 部署：生产健康检查 200；前端能调用线上 API 全流程走通。

## 假设与弹性
- 已有 HTML/CSS 基础；样式可用简易方案或 UI 库。  
- 时间不足：保持“每周 1 主练习 + 1 复盘”，必要时顺延周数，不删练习。  
- 记录：每周末提交代码与笔记到仓库，便于回顾与对比。
