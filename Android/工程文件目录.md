- <details>
  <summary>目录结构和说明</summary>
  
  ```
  ├── 项目根目录
      ├── .gradle/                  # Gradle 缓存（自动生成，无需修改）
      ├── .idea/                    # Android Studio 项目配置（IDE 设置，自动生成）
      ├── app/                      # 核心模块（开发主要区域）
      │   ├── build/                # 构建输出目录（自动生成，含 APK 和中间文件）
      │   ├── libs/                 # 本地依赖库（.jar/.aar 文件，手动放入后引用，可选）
      │   ├── src/                  # 源代码与资源根目录
      │   │   ├── androidTest/      # 仪器化测试（运行在真机/模拟器，如 UI 测试）
      │   │   ├── main/             # 主目录（正式环境代码/资源）
      │   │   │   ├── assets/       # 原始资源（如 HTML/PDF，需通过 AssetManager 读取，可选）
      │   │   │   ├── java/         # Kotlin/Java 源代码（包名结构，如 com.example.app）
      │   │   │   ├── res/          # 应用资源（编译后可直接调用）
      │   │   │   │   ├── drawable/ # 图片资源（按分辨率分 mdpi/hdpi 等子目录）
      │   │   │   │   ├── layout/   # XML 布局文件（界面结构，如 activity_main.xml）
      │   │   │   │   ├── values/   # 常量资源（strings.xml 字符串、colors.xml 颜色等）
      │   │   │   │   ├── mipmap/   # 应用图标（启动图标推荐存放处，系统自动适配分辨率）
      │   │   │   │   └── anim/     # 动画资源（补间/属性动画 XML 定义）
      │   │   │   └── AndroidManifest.xml  # 应用清单（声明组件/权限/版本，核心配置文件）
      │   │   └── test/             # 本地单元测试（运行在 JVM，纯逻辑测试）
      │   └── build.gradle          # 模块级 Gradle 配置（编译版本/依赖库等）
      ├── gradle/                   # Gradle 构建工具配置（含 wrapper，统一版本）
      ├── .gitignore                # 过滤文件
      ├── build.gradle              # 项目级 Gradle 配置（全局依赖仓库/构建规则）
      ├── gradle.properties         # 全局参数配置
      ├── gradlew                   #
      ├── gradlew.bat               # 
      ├── local.properties          # 本地环境配置（SDK 路径等，仅本地生效，不提交版本控制）
      └── settings.gradle           # 模块管理（声明项目包含的模块，如 include ':app'）
   ```
</details>


## 注意
- 各个Android Studio 切换至Project模式，查看目录结构，也许是会有差异，主要结构是一直就可以了，不要在意小细节。
- 上面的目录不用死记硬背，在日常开发慢慢熟悉即可。
- 如果有什么新的变化记得过来补充。
