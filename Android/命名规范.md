
## 资源文件 
<details>
  <summary>布局文件命名</summary>
    
  - 命名方式一(类型)
    ```
    activity_xxx.xml // activity布局
    fragment_xxx.xml // fragment布局
    dialog_xxx.xml   // dialog布局
    layout_xxx.xml   // 共用布局
    menu_xxx.xml     // 菜单
    item_xxx.xml     // 列表item

    - 优点：资源名称简洁明了。
    - 缺点：不利于资源的划分。
    - 使用：本人更趋向使用这种，因为资源划分再好看意义不大，简单实用才是王道。
    ```
    
  - 命名方式二(模块+功能+类型)
    ```
    home_activity
    home_fragment
    home_dialog
    home_share_activity
    home_share_dialog
    home_share_item_dialog
    home_share_item_list_dialog
    home_share_item_grid_dialog

    setting_activity
    setting_fragment
    setting_dialog
    setting_share_activity
    setting_share_dialog
    setting_share_item_dialog
    setting_share_item_list_dialog
    setting_share_item_grid_dialog

    - 优点：资源划分清晰。
    - 缺点：资源名称变长变复杂了。
    ```
</details>
